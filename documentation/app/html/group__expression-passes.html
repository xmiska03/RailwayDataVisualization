Detailed Description

These transform bound expressions. Some transforms utilize a guarantee, which is provided as an Expression which is guaranteed to evaluate to true. The guaranteed_true_predicate need not be bound, but canonicalization is currently deferred to producers of guarantees. For example in order to be recognized as a guarantee on a field value, an Expression must be a call to "equal" with field_ref LHS and literal RHS. Flipping the arguments, "is_in" with a one-long value_set, ... or other semantically identical Expressions will not be recognized.

Replace all named field refs (e.g. "x" or "x.y") with field paths (e.g. [0] or [1,3])

This isn't usually needed and does not offer any simplification by itself. However, it can be useful to normalize an expression to paths to make it simpler to work with.
