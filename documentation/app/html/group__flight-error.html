Detailed Description

Types for handling errors from RPCs. Flight uses a set of status codes standardized across Flight implementations, so these types let applications work directly with those codes instead of having to translate to and from Arrow Status.

Abstract status code for an RPC as per the Flight specification.

Enumerator
kOk 

No error.

kUnknown 

An unknown error occurred.

kInternal 

An error occurred in the transport implementation, or an error internal to the service implementation occurred.

kInvalidArgument 

An argument is invalid.

kTimedOut 

The request timed out.

kNotFound 

An argument is not necessarily invalid, but references some resource that does not exist. Prefer over kInvalidArgument where applicable.

kAlreadyExists 

The request attempted to create some resource that does not exist.

kCancelled 

The request was explicitly cancelled.

kUnauthenticated 

The client is not authenticated.

kUnauthorized 

The client is not authorized to perform this request.

kUnimplemented 

The request is not implemented.

kUnavailable 

There is a network connectivity error, or some resource is otherwise unavailable. Most likely a temporary condition.
